# Proposal: Privy-Jiner Open Source Migration

## Why

瑾儿私人管家系统已经成熟并验证了核心功能，现在是将其开源的最佳时机。当前系统存在三个关键问题需要解决：

1. **可访问性受限**：系统绑定在本地环境，无法让其他用户受益
2. **分发渠道单一**：仅支持 OpenClaw 插件模式，无法满足独立部署需求
3. **生态增长受限**：无法形成社区贡献和插件生态

OpenClaw 生态正在快速增长（247K+ GitHub stars），但市场上缺乏垂直领域的个人管家类插件。通过开源并采用混合模式，可以：
- 扩大用户覆盖面（独立部署 + OpenClaw 插件）
- 建立社区和贡献者生态
- 推动个人 AI 助手的最佳实践

## What Changes

### 1. 混合模式架构

- **核心独立部署**：系统可独立运行，不依赖 OpenClaw Gateway
- **OpenClaw 插件支持**：兼容 OpenClaw 插件接口，可一键安装
- **统一架构设计**：两种模式共享相同的核心代码和接口

### 2. 系统与数据分离

- **系统开源**：所有代码、配置、文档使用 MIT 许可证
- **数据本地化**：用户数据保存在本地 SQLite 数据库，不上传云端
- **隐私保护**：敏感信息（API Key、凭证）存储在本地配置文件

### 3. 模块化重构

- **核心框架提取**：Agent 协调器、任务管理、记忆管理
- **功能模块解耦**：财务、健康、时尚、知识、新闻等模块可独立使用
- **插件接口标准化**：统一的插件开发接口和生命周期管理

### 4. 数据库选型：灵活配置（独立部署模式）

- **默认选择（SQLite）**：零依赖部署，开箱即用，满足个人使用场景
- **可选扩展（MySQL/PostgreSQL）**：独立部署模式下用户可自由选择数据库类型
- **OpenClaw 插件模式**：固定使用 SQLite（确保轻量化）

**选择 SQLite 的原因（默认）：**
- **零依赖部署**：不需要安装数据库服务器，开箱即用
- **本地数据隐私**：单文件存储，完全本地化，不上传云端
- **高性能**：WAL 模式支持高并发读写，足够支撑个人使用场景
- **易于备份**：单文件数据库，备份和迁移只需复制文件
- **跨平台**：支持 macOS/Linux/Windows，适配性强

**支持其他数据库的原因（可选）：**
- **复杂场景**：多用户协作、复杂事务
- **大数据量**：数据量 > 10GB
- **高并发需求**：需要更高的并发性能
- **企业级功能**：需要数据库服务器的高级特性

**支持的数据库类型（独立部署模式）：**
- SQLite（默认）
- MySQL 8.0+
- PostgreSQL 14+
- 其他（通过插件扩展）

**选择 SQLite 的原因：**

- **零依赖部署**：不需要安装数据库服务器，开箱即用
- **本地数据隐私**：单文件存储，完全本地化，不上传云端
- **高性能**：WAL 模式支持高并发读写，足够支撑个人使用场景
- **易于备份**：单文件数据库，备份和迁移只需复制文件
- **跨平台**：支持 macOS/Linux/Windows，适配性强

**MySQL 适用场景（可选）：**

- 仅在财务管理模块需要复杂事务和多用户时才使用
- 个人使用场景下，SQLite 性能和功能完全满足需求

### 5. 双仓库维护（手动提交）

- **GitHub 主仓库**：`https://github.com/blanco-00/privy-jiner`（国际社区）
- **Gitee 镜像仓库**：`https://gitee.com/232911373/privy-jiner`（国内社区）
- **手动同步**：修改代码或文档后，手动提交到两个仓库（无自动化同步功能）

### 6. 国际化支持

- **中英文双语**：系统界面和文档支持中英文切换
- **语言配置**：用户可选择默认语言（中文/英文）
- **实时切换**：Web Dashboard 支持实时语言切换
- **i18n 框架**：使用标准国际化框架管理翻译文本

### 7. 高性能架构设计

- **SQLite + WAL 模式**：单文件数据库，支持高并发读写
- **事件驱动架构**：模块间通过事件总线解耦
- **缓存优化**：常用数据和会话状态缓存

## Impact

### Affected Code

- **核心代码**：所有 `/Users/hannah/.openclaw/workspace/projects/butler-system/` 下的代码需要重构和模块化
- **数据库设计**：从 MySQL 迁移到 SQLite（核心）+ MySQL（财务可选）
- **配置文件**：统一配置格式，支持独立模式和插件模式
- **前端界面**：Web Dashboard 需要适配两种部署模式

### Affected Dependencies

- **新增依赖**：
  - `better-sqlite3`: SQLite 数据库（核心数据存储）
  - `uuid`: 生成唯一标识符
- **保留依赖**：
  - Spring Boot 3.2.0（财务管理后端）
  - Node.js 20+（核心框架）
- **移除依赖**：
  - MySQL 8.0（完全迁移到 SQLite，个人场景足够）

### Affected Systems

- **数据迁移**：现有测试数据需要清理，生成新的示例数据
- **CI/CD**：配置 GitHub Actions，实现自动构建、测试、同步到 Gitee
- **文档体系**：重构为多级文档（用户手册、开发文档、API 文档）

### Breaking Changes

- **数据库变更**：从 MySQL 迁移到 SQLite 需要数据迁移脚本
- **配置变更**：`openclaw.json` 需要新的配置字段
- **插件接口**：OpenClaw 插件接口需要遵循新的标准
